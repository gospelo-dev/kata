# Lint ルール一覧

`gospelo-kata lint` が検出するすべてのルールコードと対処法。

---

## テンプレートモード (ソース .kata.md)

テンプレート構文の検証。`{#schema}` / `{{ }}` / `{% %}` を含むファイルで適用。

### S — スキーマ検証

| コード | レベル | 説明 |
|--------|--------|------|
| S000 | info | スキーマが定義されていない (インライン・ファイル参照とも) |
| S001 | error | スキーマ構文エラー (JSON/YAML パースに失敗) |
| S002 | error | スキーマがマッピング (オブジェクト) でない |
| S003 | warning | スキーマに `type` フィールドがない |
| S004 | error | スキーマファイルが見つからない |

**対処**: `{#schema ... #}` ブロックの YAML 構文を確認。

### T — テンプレートブロック構造

| コード | レベル | 説明 |
|--------|--------|------|
| T001 | error | `{% for %}` または `{% if %}` が閉じられていない |
| T002 | error | `{% elif %}` に対応する `{% if %}` がない |
| T003 | error | `{% else %}` に対応する `{% if %}` / `{% for %}` がない |
| T004 | error | `{% endif %}` に対応する `{% if %}` がない |
| T005 | error | `{% endfor %}` に対応する `{% for %}` がない |
| T006 | warning | 不明なブロックタグ (Jinja2 標準外) |

**対処**: `{% for %}...{% endfor %}`、`{% if %}...{% endif %}` の対応を確認。

### F — フィルター検証

| コード | レベル | 説明 |
|--------|--------|------|
| F001 | error | 不明なフィルター名 |

**対処**: 組み込みフィルター一覧は [KATA Markdown™ フォーマット](kata-markdown-format.md) を参照。

### V — 変数参照

| コード | レベル | 説明 |
|--------|--------|------|
| V001 | warning | テンプレート変数がスキーマのプロパティにない |
| V002 | info | スキーマのプロパティがテンプレートで未使用 |

**対処**: V001 — スキーマにプロパティを追加するか、変数名のタイポを修正。V002 — 情報のみ。

## ドキュメントモード (レンダリング済み .kata.md / .md)

レンダリング済みドキュメントの構造検証。`data-kata` span を持つファイルで自動適用。

### D — ドキュメント構造

| コード | レベル | 説明 |
|--------|--------|------|
| D001 | info | 外部スキーマが見つからない (インラインアンカーを使用) |
| D002 | error | 必須セクション (`## 見出し`) が欠落 |
| D003 | warning | テーブルのカラム数が不整合 |
| D004 | warning | 空のセクション (見出しのみで内容なし) |
| D005 | warning | `data-kata` アンカーがスキーマプロパティに存在しない |
| D006 | info | スキーマに定義されたがどの annotation からも参照されていないプロパティ |
| D007 | warning | enum 値が許可値リストにない |
| D008 | warning | 文字列が minLength に満たない |
| D009 | warning | 文字列が maxLength を超過 |
| D010 | warning | 配列の要素数が minItems/maxItems の範囲外 |
| D011 | warning | バッジクラスとステータスリンク値が不一致 (kata-card) |
| D012 | error/warning | kata-card 構造の問題 (未閉じ、kata-left/kata-right 欠落) |
| D014 | warning | `<details>` または `<style>` セクションが欠落 |
| D015 | warning | テーブル内の enum 値に data-kata アノテーションがない |
| D016 | error | data-kata span 内に HTML タグが含まれている |

### D016 の詳細

`data-kata` span の値は **プレーンテキスト** である必要があります。HTML タグが含まれると:
- `extract` の正規表現 (`[^<]*`) がマッチに失敗し、データが欠落する
- ブラウザ表示時の XSS リスク

**対処**:
```bash
# 自動修正
gospelo-kata fmt problematic.kata.md

# render で再生成 (自動サニタイズ)
gospelo-kata render source.kata.md -o output.kata.md
```

`render` は新規出力時に `html.escape()` で自動サニタイズします。

## ルールの無効化

特定のルールを無効化するには、ドキュメント内に以下のコメントを記述:

```html
<!-- kata-lint-disable D014 D006 -->
```

スペース区切りで複数コードを指定可能。
