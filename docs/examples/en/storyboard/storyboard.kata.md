**Prompt**

```yaml
This template generates a storyboard for a scripted narrative scene such as an onboarding flow, tutorial, or short video.
cuts is an ordered list of shots. Each cut describes one screen beat and contains speaker, scene (visual / camera direction), dialogue lines, optional narration, duration_sec, and optional audio and notes.
speaker is the character identifier used in that cut; leave empty for narration-only cuts.
scene describes what is shown on screen: character positioning, expressions, camera work, and the background visual reference.
dialogue is a list of spoken lines in the order they appear; use this when a cut has a single speaker (set by cut.speaker). Each line renders as its own row in a single bubble.
lines is an alternative to dialogue for cuts where multiple characters speak in sequence. Each entry is {speaker: <character id>, text: <line>}; each entry renders as its own bubble with the speaker's icon. The first character in the characters array is rendered as the left speaker, all others as the right speaker. When lines is present, dialogue and cut.speaker are ignored for that cut.
duration_sec is the estimated screen time in seconds; use it to pace the whole storyboard.
audio captures BGM / SFX cues. notes is free-form for directorial intent that does not belong in dialogue or scene.
image is an optional reference image for the cut (relative path or URL). image_alt is its alt text; image_width controls display width (e.g. "480").
characters defines the cast referenced by speaker ids, so readers know who "alice" or "bob" is without jumping to another file.
Each character can have an icon (relative path or URL to a small portrait); the storyboard renders the speaker icon next to their dialogue for each cut.
```
